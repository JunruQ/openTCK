# 部分设计
## 1. 项目概述
- 项目名称：openTCK
- 项目背景：TCK是风靡于合肥一中的拍手游戏，现在需要将他电子化
- 项目目标：完成游戏并提供给小范围的玩家试玩
- 受众：小范围的玩家


## 2. 功能需求
- 主要功能: 回合制战斗，技能选择，复杂技能的相互判定
- 次要功能：网络联机功能，历史战斗榜单
- 用例图

## 3. 非功能需求
- 性能要求：回合制游戏能动就行
- 可靠性
- 可用性
- 可维护性：维护到所有技能做完的有限时间
- 可扩展性：封闭的技能设计
